# Hashi.AI Dev

**An agent-native tooling lab — we build for the layer where AI stops being a chatbot and starts becoming an operator.**

---

## What we build

Hashi.AI Dev builds tools, skills, knowledge bases, and evaluation harnesses for AI agents and the humans who operate them.

Everything here starts from one question: **what does an agent actually need to operate well?**

We don't build AI apps. We build the infrastructure that makes agents useful — audit trails, knowledge bases, system doctors, skill frameworks, and agentic harnesses that hold up under real workloads.

*This is not an AI agency page. It is an open-source lab for agent-native infrastructure.*

---

## Focus areas

```
🤖  Agent-native tools     →   Built for how agents actually think and work.
```

## Why this exists

AI can generate code quickly now. The harder problem is operating agents reliably — keeping them grounded, auditable, and on target.

```
agent-native infrastructure   ·   auditable skill systems   ·   knowledge bases that agents can actually use
```

Hashi.AI Dev exists in that layer. Not to hype agents. To build things that work.

---

## Operating principles

- **Agent-native first** — every project starts from "what does the agent need?"
- **Auditable over impressive** — we show our work, not just the output
- **Small surface area** — fewer dependencies, lower trust surface, easier to maintain
- **Maintainer-led** — Hashi.AI Dev is independent; projects are maintained by me and my agents
- **OSS by default** — public tooling here is MIT licensed unless a repository states otherwise
- **Practical over performative** — real workflows, not demo reels
- **Cloud or local, bounded is best** — we deploy where it makes sense, not where it's fashionable
